# AI Governance

This document defines the guardrails, performance targets, review cadence, and rollback procedures for AI-assisted workflows in `sfdx-project-template`. It is the authoritative reference for anyone operating, extending, or disabling AI features in this repository.

---

## 1) AI Guardrails

### Scope constraints

- AI is **advisory only**. AI output (PR summary comments) is informational and never gates a merge.
- Mandatory quality/security checks — lint, LWC tests, Apex tests, CodeQL, secret scan, dependency audit — run independently and are **never bypassed** by AI workflow outcomes.
- The `ai-summary` job runs with `continue-on-error: true`; a failure or timeout does not block CI.
- AI usage is currently **limited to PR summaries** (`ai-pr-summary.yml`). Any expansion of scope (e.g., AI-assisted code fixes, auto-merge decisions) requires a governance review and an update to this document before implementation.

### Least-privilege token permissions

The workflow uses only `github.token` (repository-scoped, short-lived). No external secrets are required.

| Permission             | Level | Purpose                                     |
| ---------------------- | ----- | ------------------------------------------- |
| `models: read`         | read  | Call GitHub Models API (gpt-4o-mini)        |
| `pull-requests: write` | write | Post / update the summary comment on the PR |
| `contents: read`       | read  | Checkout repository to compute the diff     |

No elevated permissions (`admin`, `secrets`, `workflows`, `packages`) are granted.

### Data handling and sanitisation

Before any data is sent to the AI model, the following sanitisation steps are applied (see `.github/workflows/ai-pr-summary.yml`, step `Collect PR context (secrets-free)`):

- **Excluded files** (never included in diff sent to AI):
    - `package-lock.json`, `yarn.lock`, `*.lock` — dependency lock files
    - `.env`, `.env.*` — environment variable files
    - `**/*.pem`, `**/*.key`, `**/*.cert` — private keys and certificates
    - `.artifacts/**`, `IlluminatedCloud/**`, `**/*.zip`, `**/*.jar` — build artefacts and binaries
- **Diff truncation**: raw diff is truncated to **40 KB** before transmission; the Node.js step further limits to **30 KB** inside the prompt.
- **Changed file list**: only file names and status flags (`M`, `A`, `D`) are sent — no file content outside the diff.
- **No secrets in prompts**: `${{ secrets.* }}` values are never interpolated into AI prompts. The model is called with `github.token` via `Authorization: Bearer`.

### Audit trail

Every AI action is fully traceable:

- Each workflow run has a unique **run ID** visible in GitHub Actions.
- The PR comment is **upserted idempotently** (one bot comment per PR, updated on force-push) and links back to the workflow file.
- Comment footer format: `Generated by GitHub Models (gpt-4o-mini) · [Workflow](.github/workflows/ai-pr-summary.yml) · run_id traceable via Actions tab`.

### Human-in-the-loop

- All AI output is **informational**. The merge decision is always made by a human reviewer.
- The "What to Check Manually" section prompts reviewers to exercise judgment — it does not replace it.
- Reviewers must not treat AI risk level as a substitute for reading the diff.

## 2) KPIs

Baseline values must be recorded at the time of first AI workflow activation (see [Appendix: Review Log](#appendix-review-log)).

| #   | KPI                          | Definition                                                                              | Target                                       | Measurement method                                                                             |
| --- | ---------------------------- | --------------------------------------------------------------------------------------- | -------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| 1   | **PR Cycle Time**            | Time from PR opened to merged                                                           | Reduce by 20% within 3 months of AI adoption | GitHub Actions API / Insights → average PR open→merge duration                                 |
| 2   | **Review Latency**           | Time from PR opened to first human review comment                                       | < 2 hours during business hours              | GitHub Insights / `pr-metrics` label data                                                      |
| 3   | **Defect Escape Rate**       | Bugs reaching production vs. total merged PRs per sprint                                | ≤ 2% escape rate                             | Post-deploy incident count / release metrics                                                   |
| 4   | **AI Comment Accuracy**      | Ratio of AI-flagged "Check Manually" items that reviewers independently agree are valid | ≥ 70% precision                              | Quarterly manual sample of 10 closed PRs; reviewer self-report                                 |
| 5   | **AI Workflow Availability** | Percentage of PRs that receive a non-fallback AI summary comment                        | ≥ 90%                                        | `ai-summary` job success rate in GitHub Actions (filter runs where fallback was not triggered) |

## 3) Periodic AI Effectiveness Review

### Cadence

- **Quarterly** (every 3 months).
- First review: 90 days after initial AI workflow activation.

### Participants

- Repository maintainer(s)
- Tech lead

### Agenda

1. Pull KPI data for the last quarter vs. baseline (see [Appendix: Review Log](#appendix-review-log)).
2. Sample 5–10 closed PRs: read AI comments, compare against actual review outcomes — record accuracy signal for KPI 4.
3. Review any security incidents or near-misses involving the AI workflow (data handling, token misuse, prompt injection attempts).
4. Make a **keep / tune / rollback** decision:
    - **Keep**: KPIs on target, no incidents → no action required.
    - **Tune**: KPIs declining but not at rollback threshold → adjust prompt, exclusion list, or truncation limit; update this document.
    - **Rollback**: trigger threshold met (see Section 4) → follow disable/rollback procedure.

### Record-keeping

- Append an entry to the [Appendix: Review Log](#appendix-review-log) table after each review.
- If this document is updated as a result, commit with `docs: update AI governance review log`.

### Initial baseline

At the time of first AI workflow activation, record the following in the Review Log:

- Current average PR Cycle Time (last 10 PRs before activation).
- Current average Review Latency (last 10 PRs).
- Current Defect Escape Rate (last sprint).
- AI Workflow Availability: N/A (pre-activation baseline = 0%).

## 4) Disable / Rollback Procedure

### Temporary disable — single PR run

**Symptom:** AI comment is noisy, irrelevant, or revealing on a specific PR.

**Actions (either option):**

- Add a `[skip ai]` label to the PR and re-run CI.
    > ⚠️ The workflow does not natively filter on labels out of the box. To enable this, add the following condition to the `ai-summary` job in `.github/workflows/ai-pr-summary.yml`:
    >
    > ```yaml
    > if: ${{ !contains(github.event.pull_request.labels.*.name, 'skip ai') }}
    > ```
- Or manually **cancel** the `ai-summary` job mid-run via **Actions → [run] → Cancel job**.

**Recovery verification:** No bot comment posted on that PR.

---

### Emergency disable — all PRs

**Symptom:** AI workflow is causing widespread issues, incorrect output on all PRs, or a security concern has been identified.

**Check:**

```bash
# Verify the workflow file is the one running
ls -la .github/workflows/ai-pr-summary.yml
```

**Expected state:** Workflow file exists and is enabled in GitHub Settings.

**Actions (GitHub UI):**

1. Go to the repository on GitHub.
2. Navigate to **Settings → Actions → General → Workflows** (or **Actions tab → left sidebar** → select `AI PR Summary` → **···** menu).
3. Click **Disable workflow**.
4. Confirm. The workflow will no longer trigger on new PRs.

**Recovery verification:** Open a test PR; confirm no `🤖 AI PR Summary` bot comment is posted within 5 minutes.

---

### Permanent rollback

**Symptom:** AI feature is no longer providing value or a rollback trigger (see below) has been met.

**Actions:**

```bash
# 1. Create a hotfix branch
git checkout -b hotfix/<issue-id>-remove-ai-pr-summary main

# 2. Delete or rename the workflow file
rm .github/workflows/ai-pr-summary.yml
# or to archive: mv .github/workflows/ai-pr-summary.yml .github/workflows/ai-pr-summary.yml.disabled

# 3. Commit with conventional commit type
git commit -m "ci: remove AI PR summary workflow"

# 4. Push and open PR
git push origin hotfix/<issue-id>-remove-ai-pr-summary
gh pr create --base main --title "ci: remove AI PR summary workflow" --body "Closes #<issue-id>"
```

Follow the hotfix process in RUNBOOK.md Section 16 for fast-tracking the PR merge.

**Recovery verification:** Merge the PR, open a new test PR, confirm no bot comment is posted.

---

### Rollback triggers

Act immediately when **any** of the following conditions are met:

| Trigger                                      | Threshold                                                          | Action                                               |
| -------------------------------------------- | ------------------------------------------------------------------ | ---------------------------------------------------- |
| AI accuracy (KPI 4) drops consistently       | < 50% precision for 2+ consecutive quarterly reviews               | Tune prompt or permanent rollback                    |
| AI workflow causes pipeline slowdown         | Average `ai-summary` job duration > 5 minutes over a 1-week window | Investigate; if not resolvable, disable              |
| Security incident traced to AI data handling | Any confirmed leak of sensitive PR content via AI prompt           | Emergency disable immediately; follow Section 10 SLA |
| GitHub Models API deprecated or unavailable  | Persistent failure rate > 50% over 2 weeks                         | Tune or remove workflow                              |
